## 수정 포인트 1: `forge.config.js` — asar 설정

### 문제

```js
// 기존
asar: true;
```

`standalone/server.js`가 asar 아카이브 안에 묶여서
`spawn('node', [serverPath])`로 실행 불가.

### 수정

```js
// 수정 후
asar: {
  unpackDir: '{standalone,node_modules/.prisma,node_modules/@prisma}'
},
extraResource: ['./standalone'],
ignore: [/^\/standalone/]  // asar 중복 포함 방지
```

```js
  name: 'PTZController',
  setupIcon: './assets/icon.ico'
}
```

## 수정 포인트 3: `prisma/schema.prisma` — Windows 엔진 타겟 추가

### 문제

```prisma
// 기존
binaryTargets = ["native", "linux-musl-arm64-openssl-3.0.x"]
```

cloudtype(Linux)에서 빌드하면 Linux 엔진만 생성됨.
Windows EXE에서 실행 시 Windows용 `.dll` 없어서 실패.

### 수정

```prisma
// 수정 후
binaryTargets = ["native", "windows", "linux-musl-arm64-openssl-3.0.x"]
```

Windows에서 `yarn build` 시 `query_engine-windows.dll.node` 생성됨.

## 수정 포인트 4: `scripts/copy-standalone.js` — .env 및 Prisma 엔진 복사

### 문제

- `.env` 파일 미복사 → `DATABASE_URL`, `NEXTAUTH_SECRET` 없음
- `.prisma/client` 엔진 파일 미복사 → Prisma 초기화 실패
- `NEXTAUTH_URL` 미설정 → NextAuth 콜백 URL 추론 실패 → 로그인 불가

### 수정

```js
// 추가된 복사 항목
[5/5] .env → standalone/.env          (NEXTAUTH_URL 자동 추가 포함)
[6/6] .prisma/client → standalone/node_modules/.prisma/client  (엔진 .dll.node 포함)
```

`.env`에 `NEXTAUTH_URL`이 없으면 자동으로 추가:

```
NEXTAUTH_URL="http://localhost:3000"
```

## 수정 포인트 5: `electron/main.js` — 서버 경로 및 환경변수 전달

### 문제

```js
// 기존: 개발/패키징 경로 구분 없음
return path.join(__dirname, "..", "standalone");
```

### 수정

```js
// 패키징 후: extraResource 경로 사용
if (app.isPackaged) {
    return path.join(process.resourcesPath, "standalone");
}
return path.join(__dirname, "..", "standalone");
```

서버 spawn 시 필수 환경변수 명시적 전달:

```js
const serverEnv = {
    ...process.env,
    ...dotenvVars, // .env 파일 파싱 값
    PORT,
    HOSTNAME,
    NODE_ENV,
    NEXTAUTH_URL: envVars.NEXTAUTH_URL || "http://localhost:3000",
};
```

## 수정 포인트 6: `electron/main.js` — Prisma 엔진 파일 경로 (핵심)

### 문제 1: 폴더 경로 지정

```js
// 기존: 폴더를 지정 → require(폴더) 실패
PRISMA_QUERY_ENGINE_LIBRARY = ".../node_modules/.prisma/client";
```

Prisma `loadLibrary()`는 이 값을 `require()`에 직접 넘기므로
**폴더가 아닌 `.node` 파일의 전체 경로**여야 함.

### 문제 2: 플랫폼 미구분으로 Linux 엔진 선택

```js
// 기존: 알파벳 순서로 탐색 → 'l'(linux)이 'q'(windows)보다 먼저 검색됨
allFiles.find((f) => f.endsWith(".dll.node") || f.endsWith(".so.node"));
// → libquery_engine-linux...so.node 선택 → Windows에서 "not a valid Win32 application"
```

### 수정

```js
// process.platform 기반으로 플랫폼에 맞는 파일 명시적 선택
const platformEngineMap = {
  win32:  ['query_engine-windows.dll.node'],
  darwin: ['libquery_engine-darwin-arm64.dylib.node', ...],
  linux:  ['libquery_engine-linux-musl-arm64-openssl-3.0.x.so.node', ...],
};
const candidates = platformEngineMap[process.platform] || [];
let engineFile = candidates.find(name => allFiles.includes(name));
// → query_engine-windows.dll.node 정확히 선택
serverEnv.PRISMA_QUERY_ENGINE_LIBRARY = path.join(prismaClientDir, engineFile);
```

## 수정 포인트 7: `scripts/bundle-node.js` — 신규 추가

EXE 배포 환경에 Node.js가 없을 경우를 대비해
portable Node.js를 자동 다운로드하는 스크립트 추가.

```bash
node scripts/bundle-node.js
# → node-bin/node.exe 생성
# → forge.config.js extraResource에 './node-bin' 추가 필요
```

## 최종 빌드 순서

```bash
# 1. Next.js 빌드 (Windows에서 실행 필수)
cd ptzcontroller_admin
yarn install
yarn build

# 2. standalone + .env + Prisma 엔진 복사
cd ptzcontroller_desktop
npm run copy:standalone

# 3. (선택) Node.js 번들
node scripts/bundle-node.js

# 4. EXE 빌드
npm run make:win
```

## 에러 → 원인 → 수정 대응표

| 에러 메시지                                   | 원인                          | 수정 파일                       |
| --------------------------------------------- | ----------------------------- | ------------------------------- |
| `next is not recognized`                      | `yarn install` 미실행         | -                               |
| `electron-forge is not recognized`            | `npm install` 미실행          | -                               |
| `server.js not found`                         | standalone 미복사             | `copy-standalone.js`            |
| `Unable to require(.prisma/client)` (첫 번째) | asar에 standalone 묶임        | `forge.config.js`               |
| `Unable to require(.prisma/client)` (두 번째) | Windows 엔진 파일 없음        | `schema.prisma`                 |
| `Unable to require(.prisma/client)` (세 번째) | 폴더 경로 지정                | `main.js`                       |
| `is not a valid Win32 application`            | Linux 엔진을 Windows에서 로드 | `main.js`                       |
| 로그인/회원가입 실패                          | `NEXTAUTH_URL` 미설정         | `copy-standalone.js`, `main.js` |
